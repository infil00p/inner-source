PhoneGap

  • Started in 2008
  • MIT Licence
  • Hundreds of contributors
  • Started on GitHub
  • Standard Open Source project, no process necessary

Apache Cordova

  • Adobe acquired Nitobi, PhoneGap became Apache Cordova
  • Management of Nitobi, IBM and others determined where to donate
  • Apache Cordova came out of Incubation in 2012

The Apache Way

  • Decisions are made on the mailing list
  • The Project Management Committee(PMC) can determine when to release
  • The PMC has to vote on the release
  • The PMC can nominate members and members are added by the Board
  • Projects must be hosted on ASF controlled infrastructure
  • Meritocracy, Consensus, etc

What that meant for PhoneGap/Cordova

  • Decisions became much slower, and got more heated
  • Cordova had to move away from the community, using JIRA, git-web, etc
  • Angry users demanding that we use GitHub instead of ASF infra
  • Angry meetings with Management demanding that we force a release
  • Developer burnout occured

Life after Open Source

Sensei OnDevice

  • An SDK to develop ML features inside Adobe Products
  • An internal product for various Adobe products
  • Used in new proprietary (not-Open) features

Why Inner Source

  • Current team consists of three developers
  • Other teams in the company were already further along
  • Team has to support rollouts to multiple products
  • The SDK has to work with existing products build systems
  • Easier to earn buy-in support from various stakeholders

Expectations

  • Everything would live in the Internal GitHub
  • Code sharing inside the company would be seamless
  • Since only internal collaborators are working on it, things should be easy

Reality

  • Company mandated tools like JIRA cause issues to be hidden
  • Not everyone is using Corporate Git
  • Various silos depending on business unit
  • Git and Perforce repositories go out of sync w.r.t adoption
  • The Build Engineering Struggle is real

Management

  • The ASF has committees who manage the project and determine process
  • Most companies practicing Agile/Lean/Scrum have various managers
  • Critical that Management be on board
  • Inner Source may work faster than the speed of management

Releases

  • Sensei On Device SDK uses semver
  • Releases are based on client need
  • Releases are internal and go through a QA and Legal process

Things that can help for both

  • Be active where your users are
  • Using GitHub/GitCorp for everything
  • Establishing a clear, agreed upon method of contrbution
  • Being able to listen and act on the desires of your users

Takeaways

  • Getting process just-right matters
  • "Inner Source" can be more agile than Open Souce
  • "Inner Source" often moves faster than management
  • Inner Source, when done properly has the best of both worlds
